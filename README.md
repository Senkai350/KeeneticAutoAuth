# Keenetic Auto Auth



Расширение для Chrome, которое автоматически входит в веб-панель роутера Keenetic. Учётные данные хранятся локально в браузере; при открытии страницы роутера выполняется авторизация и переход в `/dashboard`.

## Текст автора

Автору было лень нажимать 2 кнопки на сайте и он решил сделать это расширение.

## Возможности

- **Автовход** — при заходе на `http://<IP-роутера>/` показывается модальное окно с индикатором загрузки, выполняется вход и редирект на панель управления.
- **Настройки в окне расширения** — IP (с портом при необходимости), логин и пароль сохраняются в `chrome.storage`.
- **Проверка из окна расширения** — кнопка «Тест» отправляет тестовый RCI-запрос и показывает результат.

## Требования
- Google Chrome (или Chromium браузеры с поддержкой Manifest V3)
- Node.js 18+ и [pnpm](https://pnpm.io/) для сборки
- Роутер Keenetic в локальной сети

## Установка расширения

1. Клонируйте репозиторий и установите зависимости:

```bash
pnpm install
```

2. Соберите проект:

```bash
pnpm build
```

3. Откройте `chrome://extensions/`, включите **Режим разработчика**.
4. Нажмите **Загрузить распакованное расширение** и укажите папку `dist/`.
5. В popup расширения введите IP, логин и пароль, нажмите **Сохранить**.
6. Откройте или обновите страницу роутера в браузере (после первого сохранения настроек).

## Настройка

### Popup расширения

| Поле   | Пример          | Описание                          |
|--------|-----------------|-----------------------------------|
| IP:Порт | `192.168.1.1`  | Адрес веб-интерфейса роутера      |
| Логин  | `admin`         | Имя пользователя                  |
| Пароль | —               | Пароль учётной записи             |


## Разработка

```bash
pnpm dev
```
### CLI-тест
проверяет авторизацию из Node (удобно для отладки без расширения).

После изменений перезагрузите расширение на `chrome://extensions/` (кнопка обновления на карточке).


### Тест из терминала (`.env`)

Скопируйте `.env.example` в `.env` и заполните переменные:

```env
KEENETIC_IP=192.168.1.1
KEENETIC_LOGIN=admin
KEENETIC_PASSWORD=ваш_пароль
```

Запуск:

```bash
pnpm test:auth
```

Опционально можно задать другой RCI-endpoint:

```env
KEENETIC_TEST_ENDPOINT=rci/show/interface/WifiMaster0
```

## Скрипты

| Команда           | Описание                                      |
|-------------------|-----------------------------------------------|
| `pnpm build`      | Сборка popup, background, content-script и page-runner в `dist/` |
| `pnpm dev`        | Сборка в режиме watch                         |

```
    ├── useKeenetic.ts
    └── ...

scripts/
└── test-auth.ts         # CLI-проверка авторизации
```

## Как это устроено

Авторизация выполняется **в контексте страницы роутера** (`page-runner.js` в MAIN world): так же, как при входе через обычный браузер.

Content script на URL из настроек показывает overlay и передаёт задачу в background; после успешного `POST /auth` страница перенаправляется на `/dashboard`.


## Безопасность

- Пароль хранится в `chrome.storage.local` только на вашем компьютере.
- Файл `.env` используется для проверки авторизации из node и добавлен в `.gitignore` — не коммитьте реальные учётные данные.
