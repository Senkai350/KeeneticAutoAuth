# Keenetic Auto Auth



Расширение для Chrome, которое автоматически входит в веб-панель роутера Keenetic. Учётные данные хранятся локально в браузере; при открытии страницы роутера выполняется авторизация и переход в `/dashboard`.

## Возможности

- **Автовход** — при заходе на `http://<IP-роутера>/` показывается модальное окно с индикатором загрузки, выполняется вход и редирект на панель управления.
- **Настройки в окне расширения** — IP (с портом при необходимости), логин и пароль сохраняются в `chrome.storage`.
- **Проверка из окна расширения** — кнопка «Тест» отправляет тестовый RCI-запрос и показывает результат.

## Требования

- Google Chrome (или другой Chromium-браузер с поддержкой Manifest V3)
- Роутер Keenetic в локальной сети
- Для сборки из исходников: Node.js 18+ и [pnpm](https://pnpm.io/)

## Установка в Google Chrome и использование

Расширение не публикуется в Chrome Web Store — его нужно установить вручную в режиме разработчика. Это обычная практика для локальных и неподписанных расширений.

### Из релиза (рекомендуется)

1. Откройте страницу [Releases](https://github.com/Senkai350/KeeneticAutoAuth/releases) репозитория и скачайте архив **`KeeneticAutoAuth.zip`**.
2. Распакуйте архив в постоянную папку, например:
   - Windows: `C:\Extensions\KeeneticAutoAuth`
   - macOS: `~/Applications/KeeneticAutoAuth`
   - Linux: `~/extensions/KeeneticAutoAuth`

   Не распаковывайте во временную папку вроде «Загрузки» — Chrome может перестать видеть расширение после очистки диска. В выбранной папке должен лежать файл `manifest.json` (если в архиве есть вложенная папка — укажите именно её при установке).

3. В Chrome откройте `chrome://extensions/`.
4. В правом верхнем углу включите **Режим разработчика**.
5. Нажмите **Загрузить распакованное расширение** и выберите распакованную папку с `manifest.json`.
6. Закрепите иконку расширения на панели инструментов (значок пазла → «Keenetic Auto Auth» → булавка).
7. Откройте popup расширения, введите IP роутера, логин и пароль, нажмите **Сохранить**.
8. Откройте или обновите страницу веб-интерфейса роутера в браузере.

После обновления до новой версии из Releases: скачайте новый `KeeneticAutoAuth.zip`, замените файлы в той же папке (или распакуйте поверх) и на `chrome://extensions/` нажмите кнопку **Обновить** на карточке расширения.

> **Важно.** Chrome периодически напоминает, что установлены расширения разработчика — это нормально. Расширение продолжит работать, пока папка на диске не удалена и оно не отключено вручную.

### Сборка из исходников

1. Клонируйте репозиторий и установите зависимости:

```bash
pnpm install
```

2. Соберите проект:

```bash
pnpm build
```

3. На `chrome://extensions/` включите **Режим разработчика**.
4. Нажмите **Загрузить распакованное расширение** и укажите папку `dist/` в корне репозитория.
5. Настройте расширение в popup (IP, логин, пароль) и откройте страницу роутера.

## Настройка

### Popup расширения

| Поле   | Пример          | Описание                          |
|--------|-----------------|-----------------------------------|
| IP:Порт | `192.168.1.1`  | Адрес веб-интерфейса роутера      |
| Логин  | `admin`         | Имя пользователя                  |
| Пароль | —               | Пароль учётной записи             |


## Разработка

```bash
pnpm dev
```
### CLI-тест
проверяет авторизацию из Node (удобно для отладки без расширения).

После изменений перезагрузите расширение на `chrome://extensions/` (кнопка обновления на карточке).


### Тест из терминала (`.env`)

Скопируйте `.env.example` в `.env` и заполните переменные:

```env
KEENETIC_IP=192.168.1.1
KEENETIC_LOGIN=admin
KEENETIC_PASSWORD=ваш_пароль
```

Запуск:

```bash
pnpm test:auth
```

Опционально можно задать другой RCI-endpoint:

```env
KEENETIC_TEST_ENDPOINT=rci/show/interface/WifiMaster0
```

```
│   ├── page-runner.ts   # код, выполняемый в контексте страницы (Origin роутера)
│   └── extension-bridge.ts
└── popup/
    ├── App.vue
    ├── useKeenetic.ts
    └── ...

scripts/
└── test-auth.ts         # CLI-проверка авторизации
```

## Как это устроено

Авторизация выполняется **в контексте страницы роутера** (`page-runner.js` в MAIN world): так же, как при входе через обычный браузер.

Content script на URL из настроек показывает overlay и передаёт задачу в background; после успешного `POST /auth` страница перенаправляется на `/dashboard`.


## Безопасность

- Пароль хранится в `chrome.storage.local` только на вашем компьютере.
- Файл `.env` используется для проверки авторизации из node и добавлен в `.gitignore` — не коммитьте реальные учётные данные.
